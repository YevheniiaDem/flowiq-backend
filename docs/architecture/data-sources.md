## Real Data vs Demo Data

### Real data paths

1. **User CRUD** — `TransactionService` creates/updates/deletes rows without seeding.
2. **CSV import** — `ImportService` parses uploaded bank files into `transactions` and `import_jobs`.
3. **Auth** — `users` table; `DemoUserSeedService` creates demo user on startup (user only, no transactions).
4. **Knowledge articles** — `knowledge_articles` seeded by Flyway V5; served via API.
5. **Tasks / notifications** — Generated by rule engines from real or seeded transactions, persisted in DB.
6. **Chat history** — `chat_conversations`, `chat_messages` persisted; replies are rule-based templates.

### Demo data paths

1. **TransactionSeedService** — When a user has **zero transactions**, first access to dashboard/analytics/forecasts/reports/AI Accountant/chat/tasks auto-inserts **6 months** of synthetic revenue/expense rows into PostgreSQL. This is **database-backed demo**, not in-memory mock.
2. **Frontend static data** — Tax profile widget, Business Guide tabs (FOP groups, taxes, KVED, recommendations), FOP eligibility checker, integrations coming-soon card.
3. **Legacy mock-data folder** — `flowiq-frontend/src/mock-data/*.ts` — mostly **unused** by live UI (superseded by API); still used for tax profile and integrations display.

### How to tell real from demo

| Signal | Meaning |
|--------|---------|
| User uploaded CSV or created transactions manually | **Real** financial data |
| User only browsed app without import/CRUD | Likely **seeded demo** in `transactions` |
| Tax profile on dashboard | **Frontend mock** regardless of backend |
| Business Guide → Articles tab | **Real** API/DB |
| Business Guide → Groups/Taxes/KVED | **Frontend mock** |

## TransactionSeedService

**Class:** `com.flowiq.service.TransactionSeedService`  
**Table:** `transactions`

### Behavior

1. **`seedIfEmpty(User user)`** — If user has no transactions:
   - Inserts 6 months of `REVENUE` and `EXPENSE` rows.
   - Monthly revenue targets (UAH): `150000, 165000, 178000, 190000, 204300, 245400`.
   - Revenue split: Online Sales 59%, Subscriptions 24%, Consulting 12%, Partnerships 5%.
   - Expenses: 62% of revenue, split across Salaries 41%, Marketing 29%, Infrastructure 12%, Operations 10%, Other 8%.
2. If user **already has data** — calls `ensureSixMonthHistory()` to backfill months with zero revenue only.

### Call sites (all invoke `seedIfEmpty`)

| Service | File | Trigger |
|---------|------|---------|
| `DashboardService` | `service/DashboardService.java` | Every dashboard endpoint (stats, insights, health, summary, cash flow, forecast snapshot) |
| `AnalyticsService` | `service/AnalyticsService.java` | All analytics methods |
| `ForecastService` | `forecasts/service/ForecastService.java` | `loadForecastData()` |
| `ReportsService` | `service/ReportsService.java` | Preview and generate |
| `ChatService` | `service/ChatService.java` | `sendMessage()` |
| `AIAccountantService` | `service/AIAccountantService.java` | `buildSnapshot()` |
| `TaskService` | `tasks/service/TaskService.java` | `ensureGeneratedTasks()` |

### Not called by

- `TransactionService` (list/CRUD)
- `ImportService`
- `NotificationService` (read path)
- `KnowledgeService`

### Impact

Any module that calls `seedIfEmpty` will **populate demo transactions on first visit**, making dashboard/analytics/forecasts appear populated. This affects architect review: metrics are **not empty-state honest** unless user has real imports.
